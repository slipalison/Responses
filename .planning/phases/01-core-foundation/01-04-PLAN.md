  <name>Task 1: Implementar ToString override para todos os Result types</name>
  <files>Responses/Result.cs</files>
  <action>
    Implementar ToString informativo para cada Result type.

    Para Result (void):
    ```csharp
    public override readonly string ToString() => IsSuccess ? "Result[Success]" : $"Result[Failed: {_error.Code}]";
    ```

    Para Result&lt;T&gt;:
    ```csharp
    public override readonly string ToString()
    {
        if (IsSuccess)
            return $"Result<Success: {_value}>";
        return $"Result<Failed: {_error.Code} - {_error.Message}>";
    }
    ```

    Para Result&lt;TValue, TError&gt;:
    ```csharp
    public override readonly string ToString()
    {
        if (IsSuccess)
            return $"Result<Success: {_value}>";
        return $"Result<Failed: {_error}>"; // Usa ToString do TError
    }
    ```

    Princípios do ToString:
    - Deve ser legível em debugger e logs
    - Incluir estado (Success/Failed) claramente
    - Incluir valor ou código de erro quando relevante
    - NÃO lançar exceção (mesmo em failure, ToString deve funcionar)
    - Manter conciso — não output JSON-like, apenas info essencial

    Cuidados:
    - _value pode ser null mesmo em success (para reference types) — handle gracefully
    - _error pode ser default em success — NÃO acessar _error.Code quando IsSuccess
  </action>
  <verify>
    Result.Ok().ToString() == "Result[Success]"
    Result.Fail("ERR", "msg").ToString() == "Result[Failed: ERR]"
    Result.Ok(42).ToString() contém "Success: 42"
  </verify>
  <done>

  </done>

<task type="auto">
  <name>Task 2: Implementar OkIf e FailIf factory methods</name>
  <files>Responses/Result.cs</files>
  <action>
    Implementar factory methods condicionais na classe Result static.

    OkIf — cria Ok SE condition é true, senão cria Fail:

    ```csharp
    // OkIf com value
    [DebuggerStepThrough]
    public static Result<T> OkIf<T>(bool condition, T value, string code, string message)
    {
        return condition ? Ok(value) : Fail<T>(code, message);
    }

    // OkIf com Error
    [DebuggerStepThrough]
    public static Result<T> OkIf<T>(bool condition, T value, Error error)
    {
        return condition ? Ok(value) : Fail<T>(error);
    }

    // OkIf para Result void
    [DebuggerStepThrough]
    public static Result OkIf(bool condition, string code, string message)
    {
        return condition ? Ok() : Fail(code, message);
    }
    ```

    FailIf — cria Fail SE condition é true, senão cria Ok:

    ```csharp
    // FailIf com error params
    [DebuggerStepThrough]
    public static Result<T> FailIf<T>(bool condition, T value, string code, string message)
    {
        return !condition ? Ok(value) : Fail<T>(code, message);
    }

    // FailIf com Error
    [DebuggerStepThrough]
    public static Result<T> FailIf<T>(bool condition, T value, Error error)
    {
        return !condition ? Ok(value) : Fail<T>(error);
    }

    // FailIf para Result void
    [DebuggerStepThrough]
    public static Result FailIf(bool condition, string code, string message)
    {
        return !condition ? Ok() : Fail(code, message);
    }
    ```

    Para Result&lt;TValue, TError&gt;:
    ```csharp
    [DebuggerStepThrough]
    public static Result<TValue, TError> OkIf<TValue, TError>(bool condition, TValue value, TError error)
        where TError : IError
    {
        return condition ? Ok<TValue, TError>(value) : Fail<TValue, TError>(error);
    }

    [DebuggerStepThrough]
    public static Result<TValue, TError> FailIf<TValue, TError>(bool condition, TValue value, TError error)
        where TError : IError
    {
        return !condition ? Ok<TValue, TError>(value) : Fail<TValue, TError>(error);
    }
    ```

    Uso esperado:
    ```csharp
    // OkIf — "ok SE condição for verdadeira"
    var result = Result.OkIf(user.Age >= 18, user, "Validation.MinorAge", "Must be 18+");

    // FailIf — "fail SE condição for verdadeira"
    var result = Result.FailIf(string.IsNullOrEmpty(email), email, "Validation.EmptyEmail", "Email required");
    ```

    [DebuggerStepThrough] em todos para não parar no debugger dentro do factory.
  </action>
  <verify>
    Result.OkIf(true, 42, "code", "msg") retorna Result.Ok(42)
    Result.OkIf(false, 42, "code", "msg") retorna Result.Fail("code", "msg")
    Result.FailIf(true, 42, "code", "msg") retorna Result.Fail("code", "msg")
    Result.FailIf(false, 42, "code", "msg") retorna Result.Ok(42)
  </verify>
  <done>

    FailIf implementado para Result, Result&lt;T&gt;, Result&lt;TValue,TError&gt;
