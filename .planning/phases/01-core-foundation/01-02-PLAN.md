  <name>Task 1: Implementar Map e Bind para Result&lt;T&gt; e Result&lt;TValue,TError&gt;</name>
  <files>Responses/Result.cs</files>
  <action>
    Implementar Map e Bind como métodos de extensão ou instance methods nos 3 structs.

    Para Result&lt;T&gt;:

    ```csharp
    // Map: transforma o valor, mantendo o estado de success/failure
    public readonly Result<TOut> Map<TOut>(Func<T, TOut> func)
    {
        return IsSuccess ? Result.Ok(func(_value)) : Result.Fail<TOut>(_error);
    }

    // Bind: encadeia operação que também pode falhar (railway-oriented)
    public readonly Result<TOut> Bind<TOut>(Func<T, Result<TOut>> func)
    {
        return IsSuccess ? func(_value) : Result.Fail<TOut>(_error);
    }
    ```

    Para Result&lt;TValue, TError&gt;:

    ```csharp
    public readonly Result<TOut, TError> Map<TOut>(Func<TValue, TOut> func)
        where TOut : notnull
    {
        return IsSuccess ? Result.Ok<TOut, TError>(func(_value)) : Result.Fail<TOut, TError>(_error);
    }

    public readonly Result<TOut, TError> Bind<TOut>(Func<TValue, Result<TOut, TError>> func)
        where TOut : notnull
    {
        return IsSuccess ? func(_value) : Result.Fail<TOut, TError>(_error);
    }
    ```

    Para Result (void):
    - Map não faz sentido para Result void (não há valor para transformar) — NÃO implementar
    - Bind pode fazer sentido para encadear operações void: `public readonly Result Bind(Func<Result> func)` — implementar se fizer sentido, senão pular

    Short-circuit behavior:
    - Quando IsSuccess é false, Map e Bind RETORNAM imediatamente com o erro original
    - NÃO executam a func passada
    - Isso é railway-oriented: o "trem" desvia para o track de erro e para

    Constraints e cuidados:
    - Manter consistência de TError entre source e result em Bind (o erro é propagado)
    - Em Map, o TOut pode ser qualquer tipo
    - Usar `readonly` em todos os métodos
    - NÃO adicionar Newtonsoft/STJ attributes ainda (Phase 2)
    - Manter compatibilidade com código existente

<task type="auto">
  <name>Task 2: Implementar Tap e Ensure para Result&lt;T&gt; e Result&lt;TValue,TError&gt;</name>
  <files>Responses/Result.cs</files>
  <action>
    Implementar Tap (side-effect) e Ensure (guard condicional).

    Para Result&lt;T&gt;:

    ```csharp
    // Tap: executa ação sem alterar o Result (logging, etc.)
    public readonly Result<T> Tap(Action<T> action)
    {
        if (IsSuccess) action(_value);
        return this;
    }

    // Ensure: valida predicado, retorna Fail se falso
    public readonly Result<T> Ensure(Predicate<T> predicate, Error error)
    {
        if (!IsSuccess) return this;
        if (!predicate(_value)) return Result.Fail<T>(error);
        return this;
    }
    ```

    Para Result&lt;TValue, TError&gt;:

    ```csharp
    public readonly Result<TValue, TError> Tap(Action<TValue> action)
    {
        if (IsSuccess) action(_value);
        return this;
    }

    public readonly Result<TValue, TError> Ensure(Predicate<TValue> predicate, TError error)
    {
        if (!IsSuccess) return this;
        if (!predicate(_value)) return Result.Fail<TValue, TError>(error);
        return this;
    }
    ```

    Tap:
    - Retorna o MESMO Result (this) sem modificação
    - Executa action APENAS se IsSuccess
    - Útil para logging, side-effects, etc.

    Ensure:
    - Se predicate é falso, retorna novo Result.Fail com o error fornecido
    - Se predicate é verdadeiro, retorna o mesmo Result
    - Se já é failure, retorna o mesmo Result sem avaliar predicate

    Para Result (void):
    - Tap faz sentido: `public readonly Result Tap(Action action)` — executa se IsSuccess
    - Ensure faz menos sentido sem valor para verificar — pode ser `Ensure(Func<bool>, Error)` para validar condições gerais. Implementar se fizer sentido.
  </action>
  <verify>
    result.Tap(x => Console.WriteLine(x)) executa em success, não executa em failure
    result.Ensure(x => x > 0, error) retorna Fail quando x <= 0
  </verify>
  <done>
    Tap implementado em Result&lt;T&gt;, Result&lt;TValue,TError&gt;, e Result (void)
    Ensure implementado em Result&lt;T&gt; e Result&lt;TValue,TError&gt;
    Side-effects não executam em failure

        var result = Result.Fail<int>("ERR", "Initial error");
