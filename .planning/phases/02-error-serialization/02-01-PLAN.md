<task type="auto">
  <name>Task 3: Criar ErrorCollection e adicionar multi-error ao Result</name>
  <files>Responses/ErrorCollection.cs, Responses/Result.cs</files>
  <action>
    Criar Responses/ErrorCollection.cs:

    ```csharp
    [StructLayout(LayoutKind.Auto)]
    public readonly struct ErrorCollection : IReadOnlyList<IError>
    {
        private readonly IError[] _errors;

        public ErrorCollection(params IError[] errors)
        {
            _errors = errors ?? Array.Empty<IError>();
        }

        public ErrorCollection(IEnumerable<IError> errors)
        {
            _errors = errors?.ToArray() ?? Array.Empty<IError>();
        }

        public int Count => _errors.Length;
        public IError this[int index] => _errors[index];
        public IEnumerator<IError> GetEnumerator() => ((IEnumerable<IError>)_errors).GetEnumerator();
        IEnumerator IEnumerable.GetEnumerator() => GetEnumerator();
    }
    ```

    Adicionar ao Result (todos os 3 types):
    - Propriedade `public ErrorCollection Errors { get; }` — retorna coleção de erros
    - Para Result void: `Errors` contém os erros diretamente
    - Para Result<T>: `Errors` contém os erros
    - Para Result<TValue, TError>: `Errors` contém os erros convertidos para IError

    Factory method adicional:
    - `Result.Fail(IEnumerable<IError> errors)` — cria Result com múltiplos erros
    - `Result.Fail<T>(IEnumerable<IError> errors)` — cria Result<T> com múltiplos erros

    No constructor interno: aceitar `ErrorCollection errors` parameter.

    Quando Error é passado como argumento único, Errors = new ErrorCollection(error).
    Quando lista é passada, Errors = new ErrorCollection(errors).
    Quando Ok, Errors = empty.
  </action>
